# 股票信息查询系统 - 完整需求文档

## 项目概述

开发一个基于Web的股票信息查询系统，提供实时股票价格查询、历史数据查看、股票搜索等功能。

## 核心功能需求

### 1. 股票搜索功能
- **描述**: 用户可以通过股票代码或公司名称搜索股票
- **需求详情**:
  - 支持模糊搜索（至少2个字符开始搜索）
  - 搜索结果实时显示（300ms防抖）
  - 显示股票代码和公司名称
  - 点击搜索结果后显示详细信息

### 2. 股票基本信息显示
- **描述**: 显示股票的基本信息卡片
- **需求详情**:
  - 股票代码和公司名称
  - 当前价格（USD）
  — 最近5年关键财报数据，包括营收，利润，毛利率，净利率，涨跌幅等数据
  - 可对股票进行分类收藏

### 3. 分析股票所在行业
- **描述**: 可对股票进行行业分析
- **需求详情**:
  - 股票公司业务所属行业
  - 对应行业的规模，以及发展速度
  — 政策是否支持，未来前景如何

### 4. 分析股票公司竞争力
  - **描述**: 可对股票公司进行分析
  - **需求详情**:
    - 企业的发展历程以及主营业务
    - 企业的商业模式以及护城河
    — 企业的文化以及管理团队情况

### 5. 分析股票投资潜力
  - **描述**: 总结最新券商研报
  - **需求详情**:
    - 根据最新券商的研报，总结投资价值

### 6. 自定义股票列表
- **描述**: 用户可以创建自定义股票关注列表
- **需求详情**:
  - 支持多个列表（如：自选股、观察列表等）
  - 列表增删改查功能
  - 列表内股票排序
  - 一键刷新所有股票数据
  - 列表数据导出（Excel/CSV）

### 7. 投资价值分析
- **描述**: 优化API调用，提高响应速度
- **需求详情**:
  - 股票基本信息缓存5分钟
  - 历史数据缓存15分钟
  - 搜索结果缓存30分钟
  - 本地存储用户自定义列表
  - 自动过期机制

### 9. 响应式设计
- **描述**: 适配不同设备屏幕
- **需求详情**:
  - 桌面端：完整功能，多列布局
  - 平板端：中等布局，功能完整
  - 手机端：单列布局，简化界面
  - 加载状态和错误处理

### 10. 可选择股票生成对应文章
  - **描述**: 基于股票价值分析生成对应文章，并发布小红书，微信公众号等平台
  - **需求详情**:
  - 对于分析过的股票可进行一键生成文章，并分享

### 11. 可展示涨跌幅最大的行业以及股票
  - **描述**: 每天动态显示今年涨跌幅最大的几个行业以及企业
  - **需求详情**:
  - 每天动态显示今年/本月，涨跌幅最大的10个行业
  - 每天动态显示今年/本月，涨跌幅最大的50个企业



## 技术需求

### 前端技术栈
- **框架**: Next.js 14 (React 18)
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **图表库**: Recharts
- **图标**: Lucide React
- **HTTP客户端**: Axios

### 后端技术栈
- **框架**: Python Flask
- **数据源**: Yahoo Finance API (yfinance)
- **数据库**: SQLite
- **CORS**: Flask-CORS
- **环境配置**: python-dotenv

### 数据库设计
- **stock_info**: 股票基本信息表
- **stock_history**: 历史数据表
- **search_history**: 搜索历史表
- **user_watchlists**: 用户自选股表
- **user_preferences**: 用户偏好设置表

### API接口设计
- `GET /api/health` - 健康检查
- `GET /api/stock/<symbol>` - 获取股票基本信息
- `GET /api/stock/<symbol>/history` - 获取股票历史数据
- `GET /api/search` - 搜索股票
- `GET /api/popular` - 获取热门股票
- `POST /api/watchlist` - 创建自选股列表
- `GET /api/watchlist/<id>` - 获取自选股列表
- `PUT /api/watchlist/<id>` - 更新自选股列表
- `DELETE /api/watchlist/<id>` - 删除自选股列表

## 设计要求

### UI/UX设计
- **设计风格**: 现代简洁，类似专业金融软件
- **主色调**: 深色模式为主，绿色（上涨）、红色（下跌）
- **字体**: 系统默认字体，确保可读性
- **布局**: 响应式网格布局，自适应不同屏幕

### 用户体验
- **加载状态**: 显示加载动画或骨架屏
- **错误处理**: 友好的错误提示
- **防抖处理**: 搜索输入300ms防抖
- **快捷操作**: 支持键盘快捷键
- **数据更新**: 自动刷新和手动刷新结合

### 性能要求
- **响应时间**: 页面加载<3秒，API响应<1秒
- **并发处理**: 支持100个并发用户
- **缓存策略**: 合理的缓存机制，减少API调用
- **数据精度**: 价格数据保留2位小数

## 部署要求

- 前端：`npm run dev` (localhost:3000)
